# TileCopy 项目计划（更新：主配置/子配置限定搜索）

> 目标：从名称清单中批量复制需要的 Tile 目录，提供可视化流程、稳定可复用的复制到目标目录；最终产出 Windows 便携 `.exe`。

## 1. 范围与原则

- 稳定优先：能跑、能恢复、可追踪日志，逐步优化。
- 可复用：配置文件可重用，可增量复制，失败可重试。
- 简化操作：用户只需选择“搜索根目录、目标目录、主配置文件”。

## 2. 配置与约束（主配置 + 子配置）

- 主配置文件：用户直接选择一个主配置文件（例如 `txt/Tile_Export.txt`）。
  - 内容：除了#开头的，每行一个“子配置文件名”（如 `Tile_Export_235-244.txt`），支持 `#`/`//` 注释与空行。
  - 位置：主配置与所有子配置文件位于同一文件夹（称为“配置目录”）。
- 子配置文件：命名形如 `Tile_Export_235-244.txt`。
  - 内容：每行一个目录名（如 `Tile_+237_+145`），同样支持注释与空行。
  - 搜索限定：该子配置仅在“对应子目录”中搜索，即搜索根目录下的 `235-244/` 子文件夹内递归匹配。
- 搜索根目录：用户单独选择。结构要求：所有子配置对应的子目录（如 `235-244/`）均直接位于“搜索根目录”下。
- 匹配规则：仅按目录 basename 精确匹配（默认大小写敏感；可后续加开关支持不敏感）。
- 忽略目录：默认跳过 `.git`、`node_modules`，可在设置中扩展。

## 3. 使用流程（用户视角）

1. 选择“搜索根目录”“目标目录”“主配置文件（包含子配置列表）”。
2. 解析主配置：读取子配置文件名；检查其在“配置目录”是否存在，缺失则提示。
3. 预扫描：
   - 对每个子配置：
     - 提取范围标签（如 `235-244`），定位“限定搜索子根目录”=`搜索根目录/235-244/`。
     - 读取该子配置中的名称，递归扫描“限定搜索子根目录”，按 basename 精确匹配。
     - 统计：匹配/目标已存在/未找到（按子配置分组）。
   - 汇总：展示总览统计与分组明细。
4. 开始复制：
   - 将所有“匹配”拷贝到“目标目录”，若已存在：默认跳过，勾选“覆盖”则强制覆盖。
5. 查看结果：
   - UI 展示复制日志与统计；写入 `YYYYMMDD-HHMMSS.log` 到 `app.getPath('userData')/logs/`。

## 4. 技术栈

- Electron 31 + Node.js 20 + TypeScript 5
- Vite + Vue 3（Composition API）
- `electron-builder` 产出 Windows portable `.exe`

## 5. 项目结构

- `src/main/`：主进程，配置解析、限定范围扫描、复制、日志、IPC。
- `src/preload/index.ts`：暴露必要 IPC（`contextIsolation: true`）。
- `src/renderer/`：Vue UI，路径选择、分组展示、执行与日志。
- `txt/`：示例主/子配置文件所在目录。

## 6. 关键实现（更新点用“★”标记）

- 配置读取 ★
  - `parseMainConfig(mainFile)`: 读取主配置，得到子配置文件名列表（去注释/空行）。
  - 校验：子配置文件是否全部存在于“配置目录”；缺失则记录警告并在 UI 提示。
- 范围解析 ★
  - `extractRangeLabel(fileName)`：从子配置文件名中解析范围标签（例如正则 `/(\d+-\d+)/` 或 `/_([\d-]+)\.txt$/`），返回如 `"235-244"`。
  - 定位限定搜索目录：`join(searchRoot, rangeLabel)`；若不存在，提示“子根目录缺失”。
- 目录扫描（稳定优先）
  - `walkDirs(root, ignoreSet)`：递归收集目录路径，仅目录，按 basename 建立唯一映射。
  - `scanInSubRoot(subRoot, names)`：仅在 `subRoot` 内匹配 `names`，输出 found/notFound，并标记目标是否存在。
  - 汇总：将各子配置结果合并，并保留分组维度（子配置名/范围标签）。
- 复制策略
  - 使用 Node 20 `fs.cp`（或 `fs-extra` 可选）复制目录。
  - 目标存在：默认跳过；可选“覆盖”。按子配置分组写入日志。
- 日志与追踪
  - 内存实时日志 + 最终写文件；命名 `YYYYMMDD-HHMMSS.log`；保留分组标记。
- 设置持久化
  - 记忆“搜索根目录/目标目录/主配置路径/覆盖开关”等到 `userData`。

## 7. UI 草图（更新）

- 路径选择：搜索根目录、目标目录、主配置文件，按钮：清空、预扫描、开始复制、打开日志目录。
- 分组展示：
  - 总览统计：匹配数、已存在数、未找到数。
  - 按子配置（文件名 + 范围标签）展开细表：匹配/未找/已存在明细。
- 状态条：运行状态 + 当前子配置 + 日志摘要。

## 8. 命令与交付

- 开发：`npm run dev`
- 构建：`npm run build`
- 打包：`npm run dist:win`（portable `.exe`）
- 使用：将主/子配置置于同一“配置目录”，搜索根目录下具备对应的范围子目录（如 `235-244/`）。

## 9. 风险与边界

- 子配置缺失或范围目录不存在：需在预扫描阶段明确提示，不进入复制。
- 范围标签解析失败：回退为“全量搜索”或直接报错（默认报错更安全）。
- 大小写敏感：默认敏感；如需不敏感需在扫描和目标存在判断处统一降维。
- 深层目录/符号链接：默认仅目录递归；不跟随符号链接以避免风险。
