# 技术决策记录

> Porter 开发过程中的关键技术选型与设计决策

---

## 1. 为什么选择 Robocopy 作为复制引擎

**决策**：使用 Windows 原生 `Robocopy` 替代 Node.js `fs.cp` / `fs-extra`。

**理由**：

- **多线程复制**：`Robocopy /MT:32` 原生支持多线程并行复制，在 SSD 和大量小文件场景下显著优于 Node.js 单线程 IO
- **操作系统级优化**：Robocopy 是 Windows 内置工具，经过数十年优化，在 Windows 文件系统上有原生的性能优势
- **丰富的复制模式**：原生支持镜像（`/MIR`）、移动（`/MOVE`）、重试（`/R`）、线程控制（`/MT`）等高级功能
- **容错能力**：内置自动重试机制（`/R:3 /W:1`），对网络路径和锁定文件有更好的处理

**权衡**：

- 仅限 Windows 平台，牺牲了跨平台能力
- 进度追踪需要额外处理（Robocopy 的 `/NP` 模式下无逐文件进度），当前通过预计算总大小 + 目录级进度来实现
- 退出码语义特殊（`< 8` 均视为成功），需要额外封装

---

## 2. 为什么选择 Electron

**决策**：使用 Electron 31 作为桌面应用框架。

**理由**：

- **文件系统访问**：需要直接访问本地文件系统进行目录扫描和复制操作，Web 应用无法满足
- **原生对话框**：使用系统原生文件/目录选择对话框，UX 更自然
- **进程隔离**：Robocopy 子进程管理、大量 IO 操作放在主进程，不阻塞 UI
- **打包分发**：`electron-builder` 可直接生成 Portable `.exe`，用户无需安装任何运行时

---

## 3. 配置解析的三种模式

**决策**：`configLoader` 支持三种配置输入方式，自动识别模式。

**理由**：

- **嵌套配置模式**（主配置+子配置）是最初为 DasViewer 导出格式设计的，保留向下兼容
- **直接清单模式**是最通用的方式，用户只需维护一个简单的目录名清单
- **目录模式**进一步降低门槛，选个目录即可，无需准备任何配置文件

**自动识别逻辑**：
1. 判断输入是文件还是目录 → 目录则进入目录模式
2. 读取文件内条目，逐一检查在配置目录下是否存在对应文件
3. 全部不存在 → 直接清单模式；部分存在 → 嵌套配置模式

---

## 4. 预检机制

**决策**：执行前强制要求预检，将扫描和匹配与实际操作分离。

**理由**：

- **避免盲操作**：大批量复制/移动/删除是高风险操作，预检可提前暴露缺失、重复等问题
- **用户确认**：预检结果展示后，用户可以审查并调整配置再执行
- **缓存复用**：预检结果在引擎中缓存，配置未变时无需重新扫描

---

## 5. 冲突策略设计

**决策**：提供三种目标已存在时的处理策略。

| 策略 | 行为 | 适用场景 |
|------|------|---------|
| 跳过已存在 | 目标存在则完全跳过 | 保守备份，保护已有数据 |
| 覆盖并合并 | 保留目标目录，逐文件覆盖 | 增量更新 |
| 先删后复制 | 清空目标目录再完整复制 | 需要目标完全重建 |

**理由**：

原始设计只有"覆盖/跳过"两态，实际使用中发现用户需求更细粒度：
- 有些场景需要保留目标中已有的额外文件（合并）
- 有些场景要求目标与源完全一致（先删后复制）

---

## 6. 动态子目录发现

**决策**：当区间目录在预期路径不存在时，可选开启在更广范围自动搜索。

**理由**：

实际使用中，源数据目录结构不总是标准的 `源根/区间标签/` 格式。可能存在中间层级，如 `源根/项目名/数据/区间标签/`。动态发现通过 BFS 在指定深度内搜索匹配的子目录名，降低用户手动调整路径的负担。

**限制**：
- 默认最大深度 4，可配置 1-8
- 仅对嵌套配置模式生效（`supportsRangeDiscovery: true`）

---

## 7. 进度追踪设计

**决策**：通过 `EventEmitter` + IPC 的方式实现主进程到渲染进程的实时进度推送。

**理由**：

- Electron 的 `ipcMain.handle` 是请求-响应模式，不适合流式进度
- 使用 `webContents.send()` 单向推送进度事件到渲染进程
- 进度数据包含：当前文件、完成百分比、速度、已处理大小、预估剩余时间

**可选增强**：
- 预检时开启"统计目录大小"可获得字节级进度，但会增加预检耗时
- 未开启时退化为目录计数级进度
