# 用户操作指南

> 适用版本：`v0.1.12`

Porter 是一个面向 Windows 的批量目录处理工具，支持按配置清单对指定目录执行**复制**、**移动**和**删除**操作，并提供预检、进度追踪和日志记录等能力。

---

## 一、适用场景与运行环境

- **适用系统**：Windows 10 / 11
- **打包格式**：Windows x64 Portable（无需安装）
- **适合场景**：
  - 根据目录名清单批量备份或迁移数据
  - 按配置文件批量复制指定文件夹
  - 在正式执行前预检，确认缺失和重复匹配
  - DasViewer 导出的瓦片选择集处理

---

## 二、准备配置数据

Porter 支持三种配置输入方式，可根据场景灵活选择。

### 1. 直接清单文件（推荐）

选择一个 `.txt` 或 `.csv` 文件作为处理清单：

- 文件内容为**一行一个目录名**
- 首行如果是表头（包含 `name`、`folder`、`path`、`目录` 等关键词），程序会自动忽略
- 支持 `#` 或 `//` 开头的行作为注释

```text
# 我的复制清单
ProjectA
ProjectB_v2
Data_2024_backup
```

### 2. 主配置 + 子配置文件

适合已有分区间组织结构的场景：

- **主配置文件**：一行一个子配置文件名
- **子配置文件**：一行一个目录名，文件名中包含区间标签（如 `Export_001-010.txt`）
- 主配置与所有子配置文件位于同一目录

### 3. 配置目录

选择一个目录作为配置来源：

- 该目录下的**一级子目录名称**会被当作待处理目录名单
- 适合"配置已经按目录组织好"的场景

## 三、界面配置

### 1. 路径配置

| 配置项 | 说明 | 示例 |
|--------|------|------|
| **主配置** | 选择文件（`.txt` / `.csv`）或目录 | `D:\projects\copy_list.txt` |
| **源目录** | 包含原始数据的根目录 | `D:\data\source` |
| **目标目录** | 复制/移动的目标位置（删除模式下不需要） | `E:\data\backup` |

> 所有路径必须为**绝对路径**。

### 2. 策略配置

#### 操作类型

- **复制**：将匹配目录复制到目标目录
- **移动**：将匹配目录移动到目标目录（源目录中的对应内容会被移走）
- **删除**：直接删除源目录中匹配到的目录（不需要目标目录）

#### 目标已存在时策略

| 策略 | 行为 | 适用场景 |
|------|------|---------|
| **跳过已存在** | 目标目录已存在时跳过 | 保守备份 |
| **覆盖并合并** | 保留目标目录，合并/更新内容 | 增量覆盖 |
| **先删除再复制** | 先清空目标目录再复制 | 完整重建 |

#### 其他选项

| 选项 | 说明 | 建议 |
|------|------|------|
| **忽略目录名大小写** | `ProjectA` 与 `projecta` 视为相同 | Windows 场景建议开启 |
| **动态发现子目录** | 在多层目录中自动搜索目标子目录 | 目录层级不固定时开启 |
| **发现最大深度** | 动态发现的搜索深度（1-8） | 推荐从 4 开始 |
| **统计目录大小** | 预检时计算每个目录的大小 | 需要速度和剩余时间估算时开启 |

### 3. 重置

点击重置后确认，将清空所有配置和结果，恢复到默认值。

## 四、操作流程

### 第一步：预检

1. 填好主配置、源目录（及目标目录）
2. 点击**预检**
3. 等待预检完成，查看结果

预检会完成以下工作：
- 解析配置文件或目录
- 识别每个区间的待处理目录名单
- 扫描源目录并匹配实际目录
- 统计缺失、重复和可处理项

**预检结果说明**：

| 状态 | 含义 |
|------|------|
| ✅ 已匹配 | 找到唯一匹配目录，可正常处理 |
| ⚠️ 缺失 | 源目录中没有找到对应目录 |
| 🔄 重复 | 找到多个同名目录，需人工确认 |

### 第二步：执行

确认预检结果后，点击执行按钮：

- **复制**：按区间逐项复制到目标目录
- **移动**：移动到目标目录，成功后源目录中对应内容被移走
- **删除**：经过两次确认后删除匹配目录（⚠️ 不可恢复）

### 第三步：查看结果

执行完成后显示结果面板：

| 标记 | 含义 |
|------|------|
| `✅ name → targetPath` | 复制/移动成功 |
| `✅ name (已删除)` | 删除成功 |
| `⏭️ name (已存在)` | 跳过（目标已存在） |
| `⚠️ name (源目录缺失)` | 源数据不存在 |
| `❌ name (错误原因)` | 处理异常 |

---

## 五、推荐使用流程

### 复制备份

1. 操作类型 → 复制；策略 → 跳过已存在
2. 预检 → 确认无大量缺失 → 执行复制

### 数据迁移

1. 操作类型 → 移动；确认目标目录正确
2. 预检 → 小批量验证后再全量执行

### 批量清理

1. 操作类型 → 删除
2. 预检 → 确认命中范围 → 两次确认后执行

---

## 六、注意事项

### 路径问题
- 所有路径必须使用绝对路径
- 删除模式下目标目录可留空

### 缺失或重复过多
检查以下配置：
- 源目录是否选错
- 是否需要开启**忽略大小写**
- 是否需要开启**动态发现子目录**
- 发现深度是否过小

### 性能建议
- 不需要速度估算时，关闭**统计目录大小**可加快预检
- 避免同时运行多个高负载复制任务
- 底层使用 `Robocopy` 多线程复制，更适合本地磁盘或常规共享目录

### 日志位置
日志存储在：`%APPDATA%\porter\logs\*.log`
