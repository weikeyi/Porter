# TileCopy 操作指南

TileCopy 用于批量复制 Tile 目录，支持预检、进度统计和多区间复制。

---

## 一、准备数据与配置文件

### 1. 使用 DasViewer 导出配置

1. 打开 **DasViewer**。
2. 点击 **“选择文件夹”**，选择数据根目录  
   （目录结构应类似：`.../Data/Tiles/...`，否则 DasViewer 解析可能异常）。
3. 使用 **“瓦片选择集”** 功能，选择需要导出的瓦片。
4. 选择完成后，点击 **“导出选择集”**，生成：
   - 一个主配置文件（通常命名为 `Tile_Export.txt`），作为 TileCopy 的“主配置文件”；
   - 多个子配置文件（按区间分拆，例如 `Tile_Export_XXXX.txt`）。

> 提示：主配置文件内容通常是一行一条「子配置文件相对路径」，TileCopy 会根据这些子配置文件自动解析每个区间的信息。

---

## 二、启动 TileCopy

1. 启动 TileCopy 应用。
2. 窗口右上方会显示运行状态：
   - 初始为：`服务就绪 (pong)`
   - 预检、复制时会显示当前操作状态和结果。

---

## 三、基础配置

主界面左侧为 **“基础配置”** 面板，依次完成以下设置：

### 1. 主配置文件

- 点击 **“选择文件”** 按钮。
- 选择通过 DasViewer 导出的主配置文件，例如：  
  `D:\projects\txt\Tile_Export.txt`
- 建议使用按钮选择，避免手工输入路径错误。

> 注意：TileCopy 要求路径为 **绝对路径**，相对路径会直接报错  
> （例如：`主配置文件路径必须为绝对路径。`）。

### 2. 源目录（Tile 根目录）

- 点击 **“选择目录”**。
- 选择包含原始 Tile 数据的根目录，例如：  
  `D:\tiles\source`
- 该目录下通常按区间或其他规则组织子目录。

### 3. 目标目录

- 点击 **“选择目录”**。
- 选择用于存放复制结果的目标根目录，例如：  
  `E:\tiles\backup`

> 建议目标目录为空或仅用于存放备份，避免与其他数据混用。

### 4. 选项

配置面板中有若干勾选项和一个数值配置：

- ☑️ **忽略目录名大小写**（默认勾选）

  - Windows 场景强烈建议保持勾选；
  - 目录匹配时会忽略大小写差异（如 `A001` 与 `a001` 视为相同）。

- ☐ **目标已存在时覆盖**（默认不勾选）

  - 勾选：目标目录中已存在同名目录时，会被新数据覆盖；
  - 不勾选：已存在的目录会被跳过，并在结果中显示为“已存在（跳过）”。

- ☐ **扫描时统计目录大小**（默认不勾选）

  - 勾选：预检阶段会统计匹配目录的体积，预检稍慢，但复制时可以显示：
    - 已传输数据量；
    - 传输速度；
    - 预计剩余时间。
  - 取消勾选：预检更快，但复制进度仅按文件个数统计，不显示字节级进度和时间估算。

- ☑️ **动态发现区间子根目录**（默认勾选）

  - 场景：实际数据目录比配置中假定的结构多了一层或几层目录，例如：  
    `源目录\某项目\A001\Tiles\...`  
    而配置只知道区间标签为 `A001`。
  - 开启后，程序会以“区间标签”为关键词，在「源目录的上一层」向下搜索对应子目录，用作实际扫描根目录。

- **发现最大深度**（数字输入，默认 `4`，范围 1–8）
  - 配合“动态发现区间子根目录”使用；
  - 表示从起始位置向下查找子目录的最大层级；
  - 如果层级过深可以适当调大，但会增加扫描时间。

### 5. 重置到初始状态

- 基础配置面板的按钮区域提供 **“重置”** 按钮。
- 点击时会弹出确认提示，确认后将：
  - 清空主配置文件、源目录、目标目录等输入；
  - 恢复所有勾选项到默认状态（包括“扫描时统计目录大小”等）；
  - 清空当前的预检结果、复制结果和进度信息。

---

## 四、预检

### 1. 执行预检

1. 确认主配置文件、源目录、目标目录都已填写，且为绝对路径；
2. 点击 **“预检”** 按钮；
3. 顶部状态栏显示 `正在预检...`，完成后更新为例如：  
   `已加载 8 个区间`。

预检会完成以下工作：

- 解析主配置文件和所有子配置文件；
- 为每个区间扫描源目录，找出应匹配的目录；
- 统计：
  - 待复制目录数量；
  - 成功匹配数；
  - 缺失数；
  - 重复匹配数。

### 2. 查看预检结果

预检完成后，中间会出现 **“预检结果”** 面板：

- 顶部汇总卡片：

  - **共计区间**：主配置中解析出的区间数量；
  - **待复制目录**：所有区间中需要处理的目录总数；
  - **缺失目录**：未在源目录中找到的目录数量；
  - **重复匹配**：同名目录在多个位置被找到的数量。

- 每个区间下面都有一个可展开的明细：
  - 标题中显示：
    - 区间标签（例如：`A001-005`）；
    - 实际用于扫描的路径（`rangeSourcePath`，可能是动态发现后的子根）。
  - 明细表格列：
    - **目录名**：配置文件中写的目录名称；
    - **状态**：
      - `已匹配`：找到唯一匹配目录；
      - `缺失`：未找到任何匹配目录；
      - `重复`：找到多个同名目录，需要人工判定；
    - **来源路径**：实际匹配到的目录路径，可能有多条；
    - **大小**：在“扫描时统计目录大小”开启时显示对应目录体积，否则为 `-`。

> 建议：在复制前确保“缺失”和“重复”数量尽量为 0。  
> 如果缺失/重复过多，优先检查：
>
> - DasViewer 导出的配置是否正确；
> - 源目录路径是否选对；
> - 是否需要调整“忽略大小写”“动态发现子根目录”等选项。

---

## 五、执行复制

### 1. 开始复制

1. 确认预检结果符合预期；
2. 点击 **“执行复制”** 按钮；
3. 状态栏显示 `正在复制...`。

复制过程中：

- 程序会逐个区间、逐个目录执行复制；
- 全局进度会被聚合展示（按全部区间合并统计）。

### 2. 复制进度显示

在基础配置面板下方会出现 **“复制进度”** 区块，包含：

- 进度百分比：整体复制完成百分比；
- 当前文件：正在处理的目录名称；
- 进度：`当前序号 / 总目录数`；
- （在统计大小时）还会显示：
  - 速度：实时传输速度，如 `120 MB/s`；
  - 已传输：`已传输 / 总大小`；
  - 剩余时间：按当前速度估算的剩余时间。

复制完成后：

- 状态栏更新为 `复制流程完成`；
- 进度条会保留一小段时间，方便查看结果。

### 3. 查看复制结果

底部 **“复制结果”** 面板按区间汇总复制情况：

- 每个区间的汇总：

  - `已复制 X`：成功复制的目录个数；
  - `已存在 Y`：因目标已有同名目录而被跳过的个数；
  - `缺失 Z`：预检阶段就缺失的目录数；
  - `失败 N`：复制过程中发生错误的目录数。

- 详情列表：
  - `✅ name → targetPath`：成功复制到目标目录；
  - `⏭️ name (已存在)`：目标中已存在，且未开启覆盖；
  - `⚠️ name (源目录缺失)`：源数据不存在；
  - `❌ name (错误原因)`：复制过程中出现异常（权限、磁盘空间等）。

---

## 六、推荐使用流程（总结）

1. **准备**：通过 DasViewer 生成主配置 + 子配置文件；
2. **配置**：在 TileCopy 中设置主配置、源目录、目标目录和选项；
3. **预检**：检查匹配情况，确认缺失/重复是否可接受；
4. **复制**：执行复制，并观察进度条与结果；
5. **验证**：必要时在目标目录检查目录数量与体积是否符合预期。

---

## 七、注意事项与故障处理

### 1. 路径与权限

- 所有路径必须是 **绝对路径**；
- 请确保当前用户对源目录与目标目录有读写权限；
- 若看到错误信息类似“权限不足”或复制失败，请尝试：
  - 检查磁盘权限；
  - 以具有足够权限的账号运行应用。

### 2. 覆盖策略

- 勾选“目标已存在时覆盖”前，请确认：
  - 目标目录中的数据可以被安全覆盖；
  - 不需要保留旧版本备份。

### 3. 性能建议

- 大量目录复制时，预检和复制都可能持续较长时间，属于正常现象；
- 若不关心体积和剩余时间，可关闭“扫描时统计目录大小”以加快预检；
- 避免同时在同一硬盘上运行多个大规模复制任务。

### 4. 常见问题排查

- **预检阶段报错**：
  - 检查主配置文件路径是否正确且为绝对路径；
  - 检查主配置文件中引用的子配置文件是否都存在。
- **某些目录一直显示“缺失”**：
  - 确认 DasViewer 导出的目录名与实际目录名一致（注意前后空格和大小写）；
  - 视情况开启/关闭“忽略目录名大小写”；
  - 检查“动态发现区间子根目录”是否需要开启，或调整“发现最大深度”。
- **复制过程中错误较多**：
  - 检查磁盘剩余空间；
  - 确认目标路径没有被系统或安全软件阻止写入。

### 5. 查看日志（高级）

- TileCopy 会在用户数据目录下记录日志，例如：  
  `C:\Users\<用户名>\AppData\Roaming\tilecopy\logs\*.log`
- 若需要反馈问题，可将对应时间的日志文件一并提供，便于排查。
