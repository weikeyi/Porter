# Porter 操作指南

> 适用版本：`v0.1.11`
> 说明：当前界面产品名为 `Porter`。仓库中仍保留部分 `TileCopy` 命名，本指南以当前界面名称和当前功能为准。

Porter 是一个面向 Windows 的批量目录处理工具，当前支持按配置对 Tile 目录执行 `复制`、`移动` 和 `删除`，并提供 `预检`、`缺失/重复检查`、`动态发现子目录`、`进度统计` 和 `日志记录` 等能力。

---

## 一、适用场景与运行环境

- 适用系统：`Windows 10 / 11`
- 当前打包目标：`Windows x64 Portable`
- 适合场景：
  - 根据 DasViewer 导出的清单批量处理 Tile 目录
  - 按目录名单批量备份、迁移或清理数据
  - 在正式执行前先做预检，确认缺失和重复匹配

---

## 二、准备配置数据

### 1. DasViewer 导出模式（主配置 + 子配置）

1. 打开 DasViewer。
2. 选择数据根目录。
3. 使用“瓦片选择集”选择需要处理的瓦片。
4. 导出选择集后，通常会得到：
   - 一个主配置文件，例如 `Tile_Export.txt`
   - 多个子配置文件，例如 `Tile_Export_A001-005.txt`

在这种模式下，主配置文件通常一行一个子配置文件相对路径，Porter 会继续读取这些子配置文件并解析区间。

### 2. 直接清单文件模式

当前版本支持直接选择一个 `.txt` 或 `.csv` 文件作为处理清单：

- 文件内容为“一行一个目录名”
- 首行如果是表头，且包含 `name`、`folder`、`path`、`目录`、`文件夹` 等关键词，程序会自动忽略这一行
- 整个文件会按“单区间清单”处理

适合已经有目录名单、但没有主配置 + 子配置结构的场景。

### 3. 配置目录模式

当前版本还支持直接选择一个目录作为配置来源：

- 选中目录后，目录下的一级子目录名称会被当作待处理目录名单
- 适合“配置已经按目录组织好”的场景

---

## 三、启动应用

1. 启动 `Porter`。
2. 顶部状态栏初始通常显示：`服务就绪 (pong)`。
3. 执行不同操作时，状态栏可能显示：
   - `正在预检...`
   - `正在处理...`
   - `正在删除...`
   - `操作流程完成`

## 四、界面配置

### 1. 路径配置

#### 主配置文件

- 可点击 `选择文件`，选择 `.txt` / `.csv` 配置文件
- 也可点击 `选择目录`，直接把一个目录作为配置列表
- 示例：
  - `D:\projects\txt\Tile_Export.txt`
  - `D:\projects\config-list`

#### 源目录（Tile 根）

- 选择包含原始数据的根目录
- 示例：`D:\tiles\source`

#### 目标目录

- 在 `复制` 或 `移动` 模式下必填
- 在 `删除` 模式下不需要，输入框会禁用
- 示例：`E:\tiles\backup`

> 注意：当前版本要求路径为绝对路径，否则会直接报错。

### 2. 策略配置

#### 操作类型

- `复制`：将匹配目录复制到目标目录
- `移动`：将匹配目录移动到目标目录，成功后源目录会被移走
- `删除`：直接删除源目录中匹配到的目录，不需要目标目录

#### 动态发现子目录

适合实际数据目录比配置中预期的路径更深的场景，例如：
`源目录\某项目\A001\Tiles\...`

而配置里只知道区间标签是 `A001`。

开启后，程序会在更上层目录范围内自动搜索对应区间目录，并把找到的目录作为实际扫描根目录。

#### 发现最大深度

- 仅在开启“动态发现子目录”时生效
- 当前范围：`1-8`
- 推荐从 `4` 开始
- 深度越大，搜索越全面，但预检会更慢

#### 忽略目录名大小写

- Windows 场景建议开启
- 开启后，`A001` 与 `a001` 会视为相同名称进行匹配

#### 目标已存在时策略

当前版本不再只有“是否覆盖”两态，而是三种策略：

- `跳过已存在`
  - 目标目录已经存在时，跳过该目录
  - 适合保守备份，不希望改动目标数据

- `覆盖并合并`
  - 保留目标目录，并把源内容合并或更新进去
  - 适合增量覆盖场景

- `先删除旧目录再复制`
  - 先删除目标中的旧目录，再执行复制
  - 适合希望目标目录被完整重建的场景
  - 风险最高，执行前建议确认目标数据可被清理

#### 扫描时统计目录大小

- 开启后，预检会更慢
- 但执行阶段可以显示：
  - 速度
  - 已处理大小
  - 预计剩余时间
- 关闭后，进度主要按目录数量统计

### 3. 重置

点击 `重置` 后会弹出确认框。确认后会：

- 清空主配置、源目录、目标目录
- 清空预检结果、执行结果和进度信息
- 将策略恢复到当前版本的重置默认值：
  - 操作类型：`复制`
  - 目标已存在时策略：`跳过已存在`
  - 忽略目录名大小写：开启
  - 扫描时统计目录大小：关闭
  - 动态发现子目录：开启
  - 发现最大深度：`4`

## 五、预检

### 1. 执行预检

1. 填好主配置、源目录，以及在 `复制 / 移动` 模式下的目标目录。
2. 点击 `预检`。
3. 等待状态栏从 `正在预检...` 更新为类似 `已加载 8 个区间` 的提示。

预检阶段会完成这些工作：

- 解析主配置文件或配置目录
- 识别每个区间的待处理目录名单
- 扫描源目录并匹配实际目录
- 统计缺失、重复和可处理项

### 2. 查看预检结果

预检完成后会显示 `预检结果` 面板。

顶部汇总区会显示：

- `共计区间`
- `待复制目录` / `待移动目录` / `待删除目录`
- `缺失目录`
- `重复匹配`

每个区间的展开项里会显示：

- 区间标签
- 实际用于扫描的路径
- 每个目录的匹配状态
- 来源路径
- 大小（仅在开启大小统计时显示）

状态说明：

- `已匹配`：只找到一个可处理目录
- `缺失`：没有找到目录
- `重复`：找到多个同名目录，需要人工确认

> 建议在正式执行前尽量把“缺失目录”和“重复匹配”降到可接受范围内。

---

## 六、执行操作

### 1. 复制

1. 完成预检后，点击 `执行复制`。
2. 程序会按区间逐项复制，并在下方显示聚合进度。
3. 完成后状态栏显示 `操作流程完成`。

### 2. 移动

1. 将“操作类型”切换为 `移动`。
2. 完成预检后，点击 `执行移动`。
3. 程序会把匹配目录移动到目标目录。
4. 成功后源目录中的对应内容会被移走。

### 3. 删除

1. 将“操作类型”切换为 `删除`。
2. 此时无需填写目标目录。
3. 完成预检后，点击 `执行删除`。
4. 程序会进行两次确认，确认后才会删除源目录中的匹配目录。

> 删除为不可恢复操作，执行前请确认数据已备份或确实不再需要。

### 4. 进度显示

执行过程中会显示 `复制进度`、`移动进度` 或 `删除进度` 区块，常见信息包括：

- 完成百分比
- 当前文件（当前正在处理的目录）
- `当前序号 / 总目录数`
- 速度
- 已处理大小
- 剩余时间

如果未开启“扫描时统计目录大小”，则不会显示完整的字节级统计信息。

### 5. 结果显示

执行完成后，下方会显示对应的结果面板：

- `复制结果`
- `移动结果`
- `删除结果`

结果中常见的汇总状态有：

- `已复制 X` / `已移动 X` / `已删除 X`
- `已存在 Y`
- `缺失 Z`
- `失败 N`

详情说明：

- `✅ name → targetPath`：复制或移动成功
- `✅ name (已删除)`：删除成功
- `⏭️ name (已存在)`：目标中已存在，且当前策略为跳过
- `⚠️ name (源目录缺失)`：源数据不存在
- `❌ name (错误原因)`：处理过程中发生异常

## 七、推荐使用流程

### 1. 复制备份

1. 选择 `复制`
2. 设置目标策略为 `跳过已存在` 或 `先删除旧目录再复制`
3. 先 `预检`
4. 确认无大量缺失或重复后执行复制

### 2. 数据迁移

1. 选择 `移动`
2. 确认目标目录正确
3. 先 `预检`
4. 小批量验证后再执行全量移动

### 3. 批量清理

1. 选择 `删除`
2. 先 `预检`
3. 再次确认扫描命中范围
4. 通过两次确认后执行删除

## 八、注意事项与故障处理

### 1. 路径问题

- 主配置路径、源目录路径、目标目录路径都应使用绝对路径
- `删除` 模式除外，目标目录可留空
- 如果配置路径不存在，程序会直接报错

### 2. 配置识别问题

- 如果你使用的是 DasViewer 导出模式，请确保主配置里引用的子配置文件都存在
- 如果你使用的是清单文件模式，请确保文件中一行只写一个目录名
- 如果首行是表头，尽量使用明显的表头文字，例如 `name`、`folder`、`目录名`

### 3. 缺失或重复匹配过多

优先检查：

- 源目录是否选错
- 是否需要开启 `忽略目录名大小写`
- 是否需要开启 `动态发现子目录`
- `发现最大深度` 是否过小
- 配置中的目录名是否存在空格、大小写或命名差异

### 4. 目标目录冲突

如果目标里已经有同名目录，请根据场景选择策略：

- 想保留旧数据：用 `跳过已存在`
- 想增量合并：用 `覆盖并合并`
- 想完全替换：用 `先删除旧目录再复制`

### 5. 性能建议

- 如果不需要速度和剩余时间估算，关闭 `扫描时统计目录大小` 可以明显加快预检
- 大批量处理时，尽量避免同时运行多个高负载复制任务
- 当前版本底层在 Windows 上使用 `Robocopy`，因此更适合本地磁盘或常规共享目录场景

### 6. 日志位置

程序会把日志写到用户数据目录下的 `logs` 子目录，常见位置例如：

- `%APPDATA%\porter\logs\*.log`

如果是正式打包后的版本，也可能显示为与应用名一致的目录。反馈问题时，建议一并提供对应时间段的日志文件。

---

## 九、快速核对清单

执行前建议确认：

- 已选择正确的主配置文件或配置目录
- 已选择正确的源目录
- `复制 / 移动` 模式下已设置目标目录
- 已确认操作类型是 `复制`、`移动` 还是 `删除`
- 已查看预检结果
- 已确认缺失和重复数量在可接受范围内
