## 目标
- 在“预检/扫描阶段”支持开关控制是否统计目录大小，以提升性能并满足不同需求。
- 当关闭统计时：
  - 预检表格的“大小”列显示为“-”。
  - 复制进度改为按“文件数量”计算百分比；隐藏速度与剩余时间。

## 代码改动
### 类型扩展
- `src/main/types.ts`
  - 在 `TileCopyJobRequest` 增加 `measureSize: boolean`（默认 true）。
  - 保持 `DirectoryMatch` 结构不变（`sizeInBytes` 为 0 表示未统计）。

### 扫描选项与逻辑
- `src/main/services/scanner.ts`
  - 在 `ScanOptions` 增加 `measureSize?: boolean`（默认 true）。
  - `collectMatches(...)` 中：
    - 根据 `measureSize` 决定是否计算 `sizeInBytes`；关闭时置 0。
  - `scanDetailRange(...)` 传递该选项。

### 引擎编排与进度聚合
- `src/main/services/tilecopyEngine.ts`
  - `preflight(request)` 调用 `scanDetailRange(detail, { ignoreCase: request.ignoreCase, measureSize: request.measureSize })`。
  - 在复制进度聚合处：
    - 如果 `globalTotalBytes === 0`，则用 `currentFileIndex / totalFiles * 100` 作为 `percentage`，`estimatedRemainingSeconds` 设为 `undefined`。
    - `speedBytesPerSecond` 也设为 `undefined` 以避免误导。

### 复制服务
- `src/main/services/copyService.ts`
  - 保持现有逻辑；当 `sizeInBytes` 为 0 时，字节进度不累计，由引擎聚合层进行“按文件数量”的百分比回退。

### 预加载与 IPC
- `src/preload/index.ts`
  - 透传扩展后的 `TileCopyJobRequest`（含 `measureSize`）。
- `src/main/index.ts`
  - IPC 处理函数无需改动逻辑，仅随类型更新接受新字段。

### 渲染层 UI
- `src/renderer/App.vue`
  - 表单区新增复选框：`扫描时统计目录大小`，绑定 `form.measureSize`（默认选中）。
  - `jobRequest` 计算属性增加 `measureSize` 字段。
  - 预检表格“大小”列：当 `match.sizeInBytes <= 0` 显示 `-`。
  - 复制进度展示：当 `copyProgress.totalBytes === 0` 时隐藏“速度”和“剩余时间”，进度条使用聚合层提供的百分比（基于文件数）。

## 交互与默认值
- 默认开启统计，维持现有体验；用户可关闭以提升预检/复制开始前的响应速度。

## 风险与验证
- 风险：关闭统计时字节级进度与 ETA 不可用；采用“文件数量百分比”替代。
- 验证：
  - 开启统计：预检大小与复制字节进度正常。
  - 关闭统计：预检大小为“-”；复制进度按文件百分比推进，速度/剩余时间隐藏；流程可完成。
