## 目标
- 当区间文件夹不在源目录的第一层时，先在源目录下（限定深度）定位到名称等于区间标签的实际子根目录，再在该子根内执行目录匹配。
- 保持大小写规则、忽略集与性能开关一致，尽量不影响现有复制逻辑。

## 设计与改动
### 选项与类型
- 扩展 `TileCopyJobRequest`：
  - `discoverRangeSubRoot: boolean`（默认 false）
  - `discoveryMaxDepth: number`（默认 2）
- 扩展 `ScanOptions`：
  - `discoverRangeSubRoot?: boolean`
  - `discoveryMaxDepth?: number`

### 动态发现逻辑（scanner.ts）
- 新增 `discoverSubRoot(startRoot: string, rangeLabel: string, opts)`：
  - 目录遍历仅看目录（`readdir({ withFileTypes: true })`），不扫描文件。
  - 大小写遵从 `ignoreCase`，忽略集沿用现有 `DEFAULT_IGNORES` + 用户扩展。
  - 找到 basename 与 `rangeLabel` 标准化后相等的目录，返回其绝对路径。
  - 多结果时选择“最浅层”优先；若同层多个，保守返回第一个并标记 `rangeConflict: true`（见报告）。
- 在 `scanDetailRange(...)` 内：
  - 计算 `startRoot = dirname(detail.rangeSourcePath)`（当前实现是 `sourceRoot/rangeLabel`，取其父即 `sourceRoot`）。
  - 若 `discoverRangeSubRoot` 为真，则尝试 `discoverSubRoot(startRoot, detail.rangeLabel, { ignoreCase, ignoreSet, discoveryMaxDepth })`。
  - 使用“发现的路径”作为实际扫描根；未找到时报告区间“子根缺失”。
  - `RangeScanReport.rangeSourcePath` 填写“实际扫描根”，并新增 `rangeConflict?: boolean`（或将冲突通过 `errors[]` 传出）。

### 引擎编排（tilecopyEngine.ts）
- `preflight(request)` 传入上述选项。
- 保持复制阶段不变：复制仍依据每个 `finding.matches[0].sourcePath`，不依赖 `rangeSourcePath`。

### 界面与交互（App.vue）
- 增加表单项：
  - 复选框“动态发现区间子根目录”。
  - 数字输入“发现最大深度”（默认 2）。
- 预检结果中显示实际 `rangeSourcePath`；若发生“同名多子根”，提示冲突（例如在区间标题旁显示“发现多个候选”）。

## 性能策略
- 发现阶段与扫描阶段均“只遍历目录”，未勾选“统计目录大小”时不做 `stat` 与文件递归。
- 受控深度（默认 2）避免全盘扫描；忽略集过滤常见大树（`.git`, `node_modules` 等）。
- 可选“Glob 扫描”模式作为后续增强：用 `fast-glob` 只枚举目录（仅在用户开启时启用）。

## 风险与处理
- 多候选同名目录：选择最浅层优先并提示冲突；必要时在 UI 提供手动选择。
- 未找到子根：报告区间“子根缺失”，该区间下所有目录标记为 `missing`。
- 性能边界：深度过大可能变慢；默认值保守，用户可调。

## 验证
- 构造源目录下多层同名区间文件夹场景，确认：
  - 能正确定位到实际子根并在其内匹配目录。
  - 未勾选“统计目录大小”时扫描不做任何文件级 I/O。
  - 冲突与缺失提示正确展示。
